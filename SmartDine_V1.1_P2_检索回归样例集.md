# SmartDine V1.1 P2 检索回归样例集

## 1. 文档说明

- 对应任务：TASK-P2-04 推荐与检索回归样例集整理
- 生成目的：为 TASK-P2-02 推荐问题策略升级、TASK-P2-03 检索低置信度与复合问题兜底提供先于实现存在的回归样例。
- 适用范围：推荐入口、`/chat` 检索命中、低置信度兜底、多意图候选提示、未知问题 fallback。
- 不适用范围：不作为完整自动化测试平台；不验证 Query Rewrite 收益；不要求当前实现立即通过全部低置信度样例；不允许为了通过样例修改 KB 数据。

## 5. 后续验收使用方式

- TASK-P2-02 使用方式：
  - 使用 `should_hit` 中的推荐菜、菜品、套餐、价格、营业、饮食需求、其他高频样例检查推荐候选是否来自 active KB。
  - 推荐接口不应继续无上限返回全部 active knowledge；默认数量、分类均衡和 fallback 可用性应结合本样例集回归。
  - 推荐类样例重点关注 `case-001` 至 `case-020` 的主题覆盖，不要求推荐接口逐条回答自然语言问题。
- TASK-P2-03 使用方式：
  - `should_hit` 类作为常规问题回归，后续实现通过率要求 ≥ 95%。
  - `ambiguous` 类应进入候选提示或澄清路径，不能只返回单一确定性业务答案；后续若实现 `candidates`，应优先覆盖这些样例。
  - `low_confidence` 类应触发低置信度或未知附加条件兜底，响应中应包含 `confidence` 与 `fallbackReason`，且不得包含 `expectNotContain` 中的确定性误答。
  - `fallback` 类不应输出任何确定性业务答案，可返回通用兜底、引导用户换一种问法或建议人工确认。
- should_hit 类通过率要求：
  - P2-03 验收中 `should_hit` 类通过率 ≥ 95%。
  - P1 高频问题不大面积回退，整体回归通过率 ≥ 90%。
- low_confidence / fallback / ambiguous 验收方式：
  - `low_confidence` 触发率目标 ≥ 80%。
  - `fallback` 类不得输出 `expectNotContain` 中的确定性业务承诺。
  - `ambiguous` 类应返回候选、澄清提示或多意图提示，不应被单一局部关键词过度命中。

## 6. 风险与注意事项

- 样例不是完整自动化测试平台，当前只提供可重复消费的数据与人工评审说明。
- 样例需要随 KB 扩展持续补充，尤其是充值 / 退款、位置 / 联系方式、第三方外卖等当前未覆盖主题。
- 不允许开发者为了通过样例修改 KB 数据；若样例与业务事实冲突，应先 review 样例本身。
- 当前 `/api/suggestions` 仍从 active knowledge 映射 `question` 和 `tags[0]`，P2-02 才处理排序、限量和分类均衡。
- 当前 `/chat` 尚未提供 `confidence` / `fallbackReason` 字段，P2-03 才实现低置信度契约，本任务不提前开发。
